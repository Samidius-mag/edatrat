# Краткое резюме ТЗ

## Концепция продукта

**ЕмПоСезону** - приложение для планирования питания, которое:
- Планирует меню на неделю с учетом сезонности
- Подстраивается под акции в магазинах
- Учитывает остатки в холодильнике
- Предлагает рецепты из имеющихся продуктов

## Ключевые отличия

1. **Интеграция с акциями магазинов** - главное конкурентное преимущество
2. **Учет сезонности** - экономия и лучшее качество продуктов
3. **Умный холодильник** - минимизация потерь продуктов
4. **Персонализация** - учет целей, ограничений, вкусов

## MVP Функционал

### Обязательно (P0)
- ✅ Профиль пользователя (семья, цели, ограничения, бюджет)
- ✅ Генерация недельного меню
- ✅ Автоматический список покупок
- ✅ Ручной ввод остатков в холодильник
- ✅ Рекомендации рецептов из остатков

### Желательно (P1)
- Базовый учет сезонности
- Перетаскивание блюд в меню
- Экспорт списка покупок

### Опционально (P2)
- Интеграция с реальными магазинами
- Аналитика экономии
- Напоминания о сроке годности

## Технический стек

- **Backend:** Node.js + Express
- **Frontend:** React + Material-UI
- **База данных:** PostgreSQL + Redis
- **Развертывание:** Docker + Ubuntu 20.04.5 LTS

## Основные модули системы

1. **Профиль питания** - настройки пользователя
2. **Сезонность** - данные о сезонных продуктах
3. **Интеграция со скидками** - API магазинов
4. **Умный холодильник** - учет остатков
5. **Динамическое меню** - недельное планирование
6. **Быстрые рецепты** - подбор из остатков
7. **Список покупок** - автоматическая генерация

## Риски

1. **Интеграция с магазинами** - API могут быть недоступны
   - Решение: начать с парсинга или фиктивных данных
2. **База рецептов** - нужна большая качественная база
   - Решение: начать с 50-100 рецептов
3. **Алгоритм генерации** - сложная логика с ограничениями
   - Решение: начать с простых правил, затем ML
